# Railway Deployment Guide - CoreInventory Backend

This guide explains how to deploy the CoreInventory backend to Railway.

## Prerequisites

1. **Railway Account**: Sign up at [railway.app](https://railway.app)
2. **GitHub Repository**: Your code should be in the `backend` branch
3. **Database**: Railway provides PostgreSQL addon

## 🚀 Quick Deploy to Railway

### Option A: Deploy Button (Recommended)
[![Deploy on Railway](https://railway.app/button.svg)](https://railway.app/template/your-template-id)

### Option B: Manual Deployment

#### Step 1: Create New Project
1. Go to [railway.app](https://railway.app)
2. Click "New Project"
3. Select "Deploy from GitHub repo"
4. Choose your repository
5. **Important**: Select `backend` branch

#### Step 2: Add PostgreSQL Database
1. In your Railway project dashboard
2. Click "New Service"
3. Select "Database" → "PostgreSQL"
4. Railway will automatically create the database

#### Step 3: Configure Environment Variables
Railway will auto-generate `DATABASE_URL`. Add these additional variables:

```env
NODE_ENV=production
JWT_SECRET=your-super-secure-jwt-secret-key-here
JWT_EXPIRES_IN=7d
PORT=3000
```

**Generate JWT Secret:**
```bash
openssl rand -base64 32
```

#### Step 4: Deploy
1. Railway will automatically detect the `railway.toml` configuration
2. It will build and deploy your backend
3. You'll get a public URL like: `https://your-app.up.railway.app`

## 🔧 Configuration Files

### railway.toml
```toml
[build]
builder = "nixpacks"
buildCommand = "cd backend && npm ci && npx prisma generate && npm run build"

[deploy]
startCommand = "cd backend && npx prisma migrate deploy && npm run start:prod"
healthcheckPath = "/api/health"
healthcheckTimeout = 100
restartPolicyType = "ON_FAILURE"
restartPolicyMaxRetries = 10
```

### Environment Variables Required
- `DATABASE_URL` (auto-generated by Railway PostgreSQL)
- `NODE_ENV=production`
- `JWT_SECRET` (generate with `openssl rand -base64 32`)
- `JWT_EXPIRES_IN=7d`
- `PORT=3000`

## 🗄️ Database Setup

Railway automatically:
1. Creates PostgreSQL database
2. Sets `DATABASE_URL` environment variable
3. Runs `prisma migrate deploy` on startup
4. Seeds database if needed

## 🔍 Monitoring & Logs

### View Logs
1. Go to your Railway project
2. Click on your service
3. Go to "Deployments" tab
4. Click on latest deployment to view logs

### Health Check
Your app includes a health check endpoint:
- URL: `https://your-app.up.railway.app/api/health`
- Should return: `{"status": "ok", "timestamp": "..."}`

## 🌐 API Endpoints

After deployment, your API will be available at:
```
https://your-app.up.railway.app/api
```

### Test Endpoints:
- `GET /api/health` - Health check
- `POST /api/auth/login` - User login
- `GET /api/products` - Get products (requires auth)
- `GET /api/dashboard` - Dashboard KPIs (requires auth)

## 🔐 CORS Configuration

Update your frontend's `VITE_API_URL` to:
```env
VITE_API_URL=https://your-app.up.railway.app/api
```

The backend is configured to accept requests from:
- `localhost:8080` (development)
- Your Vercel domain (production)

## 🚨 Troubleshooting

### Build Fails
- Check Railway build logs
- Ensure `backend/package.json` has all dependencies
- Verify Node.js version compatibility

### Database Connection Issues
- Check `DATABASE_URL` environment variable
- Ensure PostgreSQL service is running
- Check Prisma schema compatibility

### Migration Issues
```bash
# If migrations fail, you can run manually:
npx prisma migrate reset --force
npx prisma migrate deploy
npx prisma db seed
```

### Environment Variables Missing
- Ensure all required env vars are set in Railway dashboard
- Check that `JWT_SECRET` is properly generated
- Verify `NODE_ENV=production`
